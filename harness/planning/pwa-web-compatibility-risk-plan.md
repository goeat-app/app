# PWA Web Compatibility Risk Plan

## Summary

Harden the Expo React Native app for its installable PWA target by replacing or adapting native-first components that can fail, degrade, or behave inconsistently on web. This plan keeps native behavior intact and adds web-specific implementations only where needed.

Resolved items are intentionally omitted from this risk map, including the recommendations route rename and typed-route failures, because the latest `yarn typecheck` passes and active route strings now point at `/recommendations/recommendations-view`.

## Actionable Risk Map

## Test Plan

- Run static checks: `yarn typecheck`, `yarn lint`, and `yarn build:web`.
- Run PWA installability checks on local preview: manifest loads, service worker registers, icons appear, app is installable, and offline behavior works.
- Run browser smoke flows in Chrome mobile viewport and desktop: onboarding redirect, email/password sign-in, Google sign-in, home, recommendations, filters, favorites, map, restaurant details, profile, and logout.
- Run native regression checks on iOS/Android for map, location, auth persistence, bottom sheets, filters, and tab navigation.
- Add focused component/unit tests where practical for auth platform branching, location fallback, filter state, map coordinate handling, and route helper behavior.

## Assumptions

- Web map target is an interactive map using `leaflet`/`react-leaflet` with OpenStreetMap tiles and no new paid map key.
- `yarn build:web` is the canonical production PWA build command.
- Web replacements should be platform wrappers or platform-specific files, not broad rewrites of shared business logic.
- Native behavior must remain unchanged unless a current bug directly blocks shared functionality.
